# 项目约定（运维）

- `cycle_web` 服务端口固定为本机 `8888`，不要改成其它端口。
- 内网穿透端口（例如 `14082`）只是映射到本机 `8888`，不是服务实际监听端口。
- 启动/重启服务统一使用：`sh start_service.sh start|restart`（默认就是 `8888`）。
- 任何代码开发完成后，必须使用 `./start_service.sh restart` 完成服务重启，不使用其它方式重启。
- 当用户提供 `curl` 示例时，如果 URL 里是内网穿透地址（例如 `103.40.14.91:14082`），调试时忽略该 IP/端口，统一换成本机服务地址：`http://127.0.0.1:8888`。
- `curl` 里的 `Referer` 若包含内网穿透地址，也按同样规则替换为本机 `http://127.0.0.1:8888/ui/` 后再验证。
